项目进程

总结

  2. django 的基本核心是 urls.py, models.py 和 views.py

    urls.py 负责识别发送过来的 request，并把这些请求分派给 views.py 中相应的请求相应函数。而在 views.py 中函数的执行过程中，可能会调用到数据库的数据，这时候就会调用 models.py。

学习计划

  1. 阅读文档，了解 Oauth2.0 服务的原理
  2. 进行 Oauth2.0 服务的实现设计
  3. 完成阶段二：实现使用 Authorization Code模式的OAuth2.0服务

项目进程

  1. 在 django 中新建应用 oauth
  2. 实现了 Oauth2.0 服务的几个基本功能：Authorization Grant, Access Token, Verify Token

总结

  1. OAuth 2.0 是目前最流行的授权机制，用来授权第三方应用，获取用户数据。

  2. Oauth 2.0 有获得令牌的四种授权方式（authorization grant）：

    • 授权码（authorization-code）
    • 隐藏式（implicit）
    • 密码式（password）：
    • 客户端凭证（client credentials）

  3. 阶段二任务中实现的是第一种授权方式：授权码（authorization code）。其完整的授权流程主要包括以下几个步骤：

    1. A 打开 B 的授权页面
    2. 验证用户是合法的之后，生成一个 authorization code，然后通过 302 跳转的方式，把这个 code 发送给 A
    3. A 用这个 code 调用 B 的 api，获得 access token
    4. A 用 access token 访问资源

  4. 本项目中实现了使用authorization code模式的Oauth2.0服务的几个api， 详见 bangumoe 的 README.md。

学习计划

  1. 学习 OIDC (OpenID Connect) 的协议内容
  2. 实现 OIDC 身份认证（核心部分）

项目进程

  1. 完善了 OAuth2.0 ，加入了用户授权的部分
  2. 实现了 OIDC 的核心部分的大部分内容，Userinfo Request 的部分还未实现，留做习题，预计明天完成

总结

今天的学习过程和前几个阶段差不多，第一眼接触到全新的协议也比较懵，于是选择了阅读文档博客园，大概花了一上午的时间（感觉在课上干这些效率不是那么高？）。

在基本理解协议内容后，感觉 OIDC 的实现基本还是可做的，很大程度上都是基于 Oauth 的内容，因此在细节方面，以及 api 的调用方面（进行了比较多的摸索）进行修改，OIDC 基本上就实现了。

另外说说 OIDC 和 OAuth 的区别（个人理解），Oauth 主要解决了授权方面的服务的问题，但无法进行身份认证，OIDC 在 Oauth 的基础之上增加了用户认证的功能，主要是引入 id token ，从而将用户信息传给客户端，实现身份认证的目的。

附：总的来说，实习四天下来，感觉收获还是非常大的，从一开始的两眼茫然，完全不觉得自己能做出来，到逐渐习惯查文档、搜罗资料，理解之后再逐步进行实现的过程，发现当把一个事情分解成很多小任务逐一完成之后，就变得容易了很多。现在回头看这几天的成果，不知不觉已经实现了很多东西，还是很有成就感的。有一个缺点就是，因为这几天是速成，加上之前也没有任何工程经验，且时间也不允许去阅读很多其他人的工程代码，所以很多是靠着自己的理解来实现，可能代码质量不是很高，而且也带来了很多之前不好的编程习惯（但是至少能跑哈哈哈）。
